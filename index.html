客户端存储

客户端存储有三种方式：cookie、localStorage、sessionStorage。

⚠️此处区别于客户端缓存，缓存存储的是资源文件，这三个方式存放的是一些键值对的文本信息。

cookie的详细概念在cookie章节中查看。

cookie和两个Storage的区别主要是cookie有自定义的失效时间，且cookie是带path的，即可以限制cookie只属于某个域下的某个路径，且cookie的大小最大不过4k。

cookie一般是由服务端返回给客户端的，之后客户端每次请求服务端时，都会在请求头里携带cookie。

区别

接下来主要细说localStorage、sessionStorge。

HTML5 web 存储：HTML5 提供了两种在客户端存储数据的新方式：localStorage 和 sessionStorage ，两者都是仅在客户端（即浏览器）中保存，不参与和服务器的通信。

早些时候，本地存储使用的是 cookie，。但是 cookie 不适合大量数据的存储，后来 HTML5 提供了 localStorage 和 sessionStorage ，这些数据不会被保存在服务器上，它也可以存储大量的数据，而不影响网站的性能。数据以键/值对存在且只允许该网页访问使用。

sessionStorage

sessionStorage用于本地存储一个会话（session）中的数据，这些数据只有在同一个会话（同一个tab标签页）中的页面才能访问，不同tab页是无法共享的。
并且当会话结束后数据也随之销毁。因此sessionStorage不是一种持久化的本地存储，仅、是会话级别的存储。只允许同一窗口访问。

⚠️只要在同一会话（tab页）下，即使刷新和跳转其他页面，sessionStorage的值也是会带过去的，只不过新旧sessionStorage没有什么联系了。

localStorage

localStorage 用于长久保存整个网站的数据，保存的数据没有过期时间，除非手动删除，并且在同源下可以读取并修改localStorage的值。

操作

  • 保存数据：localStorage.setItem(key,value) 也可用 localStorage.key = value;
  • 读取数据：localStorage.getItem(key);
  • 删除单个数据：localStorage.removeItem(key);
  • 删除所有数据：localStorage.clear();

⚠️当我们想在 localStorage 和 sessionStorage 中存储对象时，可以先用 JSON.stringify(obj) 方法将对象转换为字符串，然后进行存储。要想将值作为对象输出可以用 JSON.parse(str) 方法将字符串转换为对象。

年少意难平

创建时间2021-02-25更新时间2021-02-25字数统计329阅读时间1m

今天看到电视上，郑钧说，浪漫是时间沉淀出来的。

我承认这句话有触动到我

浪漫的漫是漫漫的、绵绵的。

这种感觉才是浪漫，如果是一瞬间的，那也激情，但是昙花一现，经不住考验，没有味道

年少时还总是体验不到绵绵漫漫，只想更快、再快、愈快

所以啊，经常会仰望其他不属于自己的东西。而对身边的东西，对自己，却少了包容和爱。

因而总是感觉缺了点什么。

总是为了说点什么而说点什么，欲赋新词强说愁。

但这可能正是青春，只是在这漫长的路上，可能更需要的是关注脚下的每一步，因为我们都值得拥有更好的。

那些真正美好的东西，是经过时间的沉淀而筛选出来的。就像酒，越酿越香。

所以不要急呀，在一切美好的东西都还没到来的时候，只需要慢慢的走，然后等它出现就好了。而此时的所有一切，身边的一切，包括自己，就是最美好的一切。

永远相信自己，相信现在。无所谓远方，无所谓明天，眼前的就是最好的。
